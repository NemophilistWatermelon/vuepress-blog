# Git 提交规范

feat：新功能（feature）
fix：修补bug
docs：文档（documentation）
Style： 格式（不影响代码运行的变动）
Refactor：重构（即不是新增功能，也不是修改bug的代码变动）
Test：增加测试
Chore：构建过程或辅助工具的变动
